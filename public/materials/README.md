## ğŸ’¡ Problem-Solving Approach

### **For Every Problem:**

1. **Understand** - Read 2-3 times, identify inputs/outputs
2. **Examples** - Create test cases (including edge cases)
3. **Brute Force** - Think simplest solution first
4. **Optimize** - Identify patterns, use right data structures
5. **Code** - Write clean, commented code
6. **Test** - Trace through examples
7. **Review** - Learn alternative approaches

**Budget: 45-60 minutes per problem**
